1.mobile part

##Newsinfo to Profile

If the user don’t login the account, all of the right top in the webdetail page will be the “reg”. and if the user logined the account, the unregistered events of the right top in the webdetail page will be the “reg”.

If the user don’t login the account, the registered events of the right top in the webdetail page will be the “reg”. And if the user click the “reg”, the page will go to login page.

##Login
In the login page, password can be hided by this code that is passwordMask=”true”.

After the user logins the account, the system will remind the user logining successfully.

##Logout
If the log.title==”Logout”,after click logout the account.

##Register
If you have no account,you can register firstly.Add a optiondialog here to choose department.

##Registered Event

this page showed all of the events the user has registered.

##Register and Unregister Event

###Register Event

If the user logined the account, he can click the “reg” button to register the event, and then, there is an alert dialog to make sure that you want to register the event. If you cancel, there is an alert to remind you. If you click Yes, you can see the new registered event in the myregistered event

###Unregister Event

At first, the user should login the account, he can go to profile page, and go to myRegistered Event, and go to the newsinfo. click the unreg button to unregister the event, and then, there is an alert dialog to make sure that you want to unregister the event. If you cancel, there is an alert to remind you. If you click Yes, the registered event in the myregistered event will be gone.

2.web part

##Navigate Bar

###Login page
In the login page, there are only three navigate bar including home, search and login, and I change the style of the navigate bar.

###staff management

In the staff page, there are 6 navigate bar including home, search, create, admin, logout, and Hi, staff.

###student management

In the staff page, there are 5 navigate bar including home, search, my registered event, logout, and Hi, student.

###admin management

In the admin page, there are 7 navigate bar including home, search, create, admin, manage, logout, and Hi, admin.

##Register user
Staff and student can register,use option to select different department.

##Manage

Manage is one of extra function. At first, the staff can only own department event.

The admin can give the authority to the staff, not for the student. The staff who have the admin authority can management all of the events, but the staff cannot give the admin authority to others.

##Quota full

Register one event,the quato of this event will be minus one.Unregister,the quato will be plus one.If quato =0,this event will can’t be registered.

##Create unique event

There is not two different events who have the same venue, date and time, so the problem happened, the system will be send the error message and refuse to create the things.
